# doctrine authoring rules

Concepts here are `type: Doctrine`: durable operating philosophy, not decisions or
facts. Shared schema is in the parent [../CLAUDE.md](../CLAUDE.md).

- `type` is always `Doctrine`. No `status`, no `resource`.
- A doctrine concept states a principle, then `Why` and `How` it applies.
- Doctrine is stable. A change here is a real shift in how the project operates,
  so it warrants a `log.md` entry and usually a backing `Decision`.
